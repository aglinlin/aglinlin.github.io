学习搭建博客

2020-09-03

通过 GitHub+Gridea+Vercel 打造属于自己的免费博客。

#前言
因为种种原因，许多写手太太纷纷前往 GitHub 上面放文。然而我看到一部分太太只是新建了一个仓库，然后把文字直接以 txt/md 的格式上传，或者在 Issues/Wiki 里面提交。这类方式虽然快捷简单，但对读者的阅读体验感并不算友好，而且并没有利用好 GitHub 这个平台。

在这里分享一下利用 GitHub 搭建一个属于自己的博客的方法。很小白，只要会打字且有一台电脑就能操作！不骗人！

博客的好处

博客完全属于你自己！不会有屏蔽词来限制你的写作，支持评论，而且相比 Wordpress 无广告，阅读体验更佳！

缺点嘛······就是没备案的博客可能会被墙，而且通常需要搭配另一个社交平台才能吸引到读者。

前期准备

准备一个供注册使用的邮箱📫
这里不推荐使用国内的邮箱。因为现在国内的邮箱基本都需要绑定手机号码，直接和个人信息挂钩，而且国内邮箱可能不支持部分外国网站的注册（如Wordpress）。
推荐大家使用无需手机验证注册的加密邮箱，如 ProtonMail和 Tutanota。我个人使用的是前者，免费版已经足够用了。
使用上面的邮箱注册 GitHub 账号
这里同时建议大家下载 Chrome 浏览器。无广告而且自带翻译功能。如果看 GitHub 的英文有些头疼的话，可以通过 Chrome 将整个网站翻译成中文。
注意！你的博客原始域名会是 你的 GitHub 用户名.github.io，所以在输入用户名时最好谨慎一些！
前往 Vercel，利用 GitHub 账号关联注册
是的，这也是一个英文网站。
Vercel 能够帮助我们自动部署 GitHub 上的项目，加速博客，而且免费提供 .now.sh 的二级域名。
新建 GitHub 仓库
进入 GitHub，点击右上角的加号，选择第一行的 New repository。在 Repository name 中输入 你的 GitHub 用户名.github.io，然后直接点击下方绿色的 Create repository。

Gridea 搭建博客

我选择的博客搭建工具是 Gridea，对小白十分友好。我们接下来进入 官网 下载 Gridea。国内推荐使用码云（Gitee）下载，速度会快一些，但个人感觉没快多少。

完成后打开 Gridea，前往 远程，然后根据官方文档 快速上手的操作一步步走下去，就部署完成了👌

用 Vercel 加速博客

搭建完成后，前往 你的 GitHub 用户名.github.io 就可以看到自己的博客了。但最近 .github.io 的网站似乎都打不开，而且即使能打开速度也很慢🐌

这时候我们可以利用 Vercel来加速自己的博客🚀

在 Vercel 上选择黑色实心按钮的 Import Project，然后选择 Import Git Repository，点击 Continue。

接下来会蹦出一个输入框，将你的博客在 GitHub 上的仓库 URL 地址复制粘贴上去，点击 Continue。

接下来继续直接点 Continue。之后你可以给自己的项目起名，比如xx的博客之类。起好名后点击 Deploy。

Vercel 现在开始自动部署了。稍等片刻后会弹出一个 Congratulations!，这表示你的项目部署成功，点击 Visit 看看速度是不是快了些。

免费获得二级域名 .now.sh

进入你的博客在 Vercel 上面的项目，然后点击顶部的 Settings，点击右侧的 Domains。

Vercel 默认分配给你的博客的域名是 你的 GitHub 用户名.vercel.app，这里我们可以在上方的输入框中，输入 你喜欢的域名.now.sh，点击 Add，如果未被使用的话可以直接添加上去。

完成后可以点击 .vercel.app 里的 Edit，将它 Redirect to（重定向） 到你新添加的域名上，最后不要忘记 Save。

然后返回Gridea 远程设置域名为 你喜欢的域名.now.sh 或你喜欢的域名.vercel.app

完成啦，撒花！目前来看 now.sh 和 .vercel.app 在国内是能够正常访问的，速度也不错。
